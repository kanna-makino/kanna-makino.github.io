超高速（弱智）开发工具——Kintone

12-27-2019
学习笔记

11月份开始进入一个“太平洋建筑”的项目。预料之中，这个项目也用了和去年那个项目类似的开发工具，叫做“Kintone”，一个日本公司开发的工具。有个中国官网，但是网上一查，还是日本鬼子用的最多，想靠这种应对劳动力减少的弱智工具赚世界第一人口大国的钱，鬼子们真是想得太多。

据说是为了降低开发门槛，让即使在技术方面一无所知只要对业务了解的人也能做开发。我翻译了一下：让不懂技术的人继续不懂技术、让懂点技术的人也变得不懂技术。


公司的技术趋势

去年七月份入职，刚好赶上公司开了一个“超高速”开发的项目，从头到尾做了九个多月的时间，一直从开发到单元测试再到系统测试。从那之后，“超高速”开发就成了公司的发展方向，越来越多的引入相关工具。公司的现状以及发展方向成了我离职的最大考虑因素。如果想要在技术上更上一层楼，那仅仅停留在会使用工具这个层次，是永远达不到你的理想的。除非你打算在这个公司工作到它倒闭或者是你老死，不然留下绝对不是最好的选择。

我很庆幸刚好赶上年底的这个契机。


领导的想法

项目的领导对这个工具赞不绝口，总是会跟我们说这个工具多么高效多么便利。我一直担心是自己了解得不够深入，不敢做出自己的评价。后来想了想，一个在这个行业已经做了十多年还问我“js怎么当前时间转换成YYYY-MM-DD的字符串”这种问题的人，想必一路走来也没写过几行代码，就我近两个月的观察，他每天的工作内容不过是分配任务、改一下计划完成日以及每天和旁边的女生分享他自认为“辉煌”的过去罢了。

能让毫无开发经验的人也能上手的开发工具，换句话说有开发经验的人用上一段时间也会变得没有开发经验。确实，你只需要会声明一个变量，会给变量赋值，至于那些高深的知识，你不用了解，因为你了解了也用不上。


存取数据

这个弱智工具最大的特点应该就是没有数据库了吧，据说是为了让公司的人在哪儿都能访问数据。我其实有点不太理解，难道用了MySQL、SQL Server的项目就无法在公司之外访问数据了？有这种疑问，大概是我了解得少的原因吧。没有数据库真的很难受，刚进入这个项目的时候，本来想找个数据库设计书看看，大概了解一下业务，结果没有。每个页面上的所有数据项大概对应一张表的样子，但是各个页面之间的关联关系，是一对一还是一对多还是多对多，并没有白字黑字的定义。那写代码了想知道这个关系怎么办，只能问了解的人，想想就觉得弱智。


数据的安全性

数据都存在这个工具的运营方的服务器上，项目的域名也是工具运营方提供的。按照我的理解，应该是所有的数据在工具提供方来看都是透明的，如果真是这样的话，那客户业务里那些机密非机密数据还有区别吗？对本公司员工分机密对其他的公司的人却全部可见？谁能保证工具开发方即将要跑路的程序员不会把这些数据也打包带走然后泄漏出去？？

因为整个项目都是js实现的，根据权限不同可能会把某个详情页面某些数据项给隐藏掉，不过仅仅接收到数据之后隐藏掉HTML元素。在HTTP请求数据的时候，参数只有查询条件并没有权限验证，也就是说，只要使用这个系统的人稍微懂点JavaScript知识的话，整个项目产生的数据对他来说就是全部可见的。

另外就是页面上某些项目的可编辑和只读，按钮的disabled都是js控制的，随便在控制台里加几行代码就能随心所欲、为所欲为，不过比较讽刺的是，一个项目一群级别比我高一级两级甚至三级，工作了十来年的人，竟然不会控制台调试，神仙。

如果我理解没错的话，那客户花了这么多钱买这么个垃圾玩意，真替他们不值。


严重的性能问题

数据存在其他公司的服务器上还会导致严重的性能问题。正常项目中的一次数据库查询操作，在这里变成了一个HTTP请求，相比于一次HTTP请求200ms~300ms，一次数据库操作所花的时间那几乎是微乎其微了，更何况数据库还有池、缓存这些技术可用。前两天做了一个Excel文件的下载，统计一个月整个公司项目的进度状况，很简单的几个操作却花了20分钟才收集完所有数据。

不支持表连接。当需要多个表的数据的时候，请求量会暴增，尤其是一些比我级别高的人Developer2的人的代码里，还在for循环里请求数据，看着都觉得心疼。

可能是出于吞吐量的方便的考虑，一次请求的数据量默认只有100条，最大不能超过500条。目前所有的功能数据量都按照小于500条去取，这个假设也是让我摸不着头脑，真是世界之大，无奇不有。


同事的现状

我也不知道公司到底是缺人到了什么地步，招了一个研究生过来，连 int a = 0; 都不会，使用变量之前要赋值都不懂。最重要的是因为学历比我高，所以工资也比我高。每天悠闲到无聊的领导可能觉得还挺开心，终于有点事情做了，每天花上3、4个小时给女研究生从声明变量开始讲起。每次他们一开始，我就觉得心里无比痛苦，觉得自己是个废物，每次都想拔出水果刀朝自己来几刀，虽然最终还是选择去阳台缓一缓。


久违的离职邮件

从3月份到9月份走了很多人，几乎每周五都能收到离职邮件，但是似乎因为到了年底大家都想捞个年终奖的缘故，似乎很久没有收到离职邮件了。

下周就要离职了，我终于可以发出这封已经想好很久的离职邮件了。
